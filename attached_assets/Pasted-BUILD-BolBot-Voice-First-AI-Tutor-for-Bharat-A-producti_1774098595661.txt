═══════════════════════════════════════════════════════
PRODUCT VISION
═══════════════════════════════════════════════════════

BolBot is the first voice-native AI tutor built 
specifically for Hindi-medium students in Tier 2/3 
India. Students in Bihar, UP, MP, and Rajasthan can 
send a voice note, photo, or text in Hindi/Hinglish/
Bhojpuri and get a warm, clear explanation back — 
like a smart elder sibling who always has time for them.

The product runs on three platforms simultaneously:
- WhatsApp (via Twilio) — primary channel
- Telegram (via python-telegram-bot) — secondary
- Web app (Flask + HTML) — for desktop users

FEATURE 1: MULTI-PLATFORM MESSAGE HANDLING

WhatsApp handler (Twilio):
- POST /webhook/whatsapp receives all message types
- Handles text, voice note, image
- Sends back text or voice note response

Telegram handler (python-telegram-bot):
- Handles /start command
- Handles text messages
- Handles voice messages (OGG format)
- Handles photo messages
- Sends back text or voice response

Web app handler (Flask):
- Clean chat interface at /
- Supports text input
- Supports microphone recording (Web Audio API)
- Supports file/image upload
- Real-time response display
- Works on mobile browser

FEATURE 2: VOICE INPUT + HINDI VOICE OUTPUT

Input:
- WhatsApp voice notes (OGG/MP3)
- Telegram voice messages (OGG)
- Web app microphone recording (WebM/WAV)
- Transcribe ALL audio using Gemini 1.5 Flash
  (natively handles Hindi audio)

Output:
- Convert BolBot text response to Hindi speech
- Use Google Cloud Text-to-Speech
  Voice: hi-IN-Wavenet-D (natural male Hindi)
  Speaking rate: 0.9 (slightly slower, clearer)
- Send audio back as voice note on WhatsApp/Telegram
- Play audio automatically on web app
- Delete temp audio files after sending

FEATURE 3: PHOTO/IMAGE EXPLANATION

- Accept photo of textbook page on all 3 platforms

FEATURE 5: CHAT MEMORY (CONVERSATION CONTEXT)

═══════════════════════════════════════════════════════
DATABASE SCHEMA (PostgreSQL on Replit)
═══════════════════════════════════════════════════════

Table: users
- id (serial primary key)
- platform (whatsapp/telegram/web)
- platform_user_id (string, unique per platform)
- phone_last4 (string, nullable)
- username (string, nullable)
- avatar_url (string, nullable)
- points (integer, default 0)
- streak_count (integer, default 0)
- last_active_date (date)
- total_messages (integer, default 0)
- badges (text array)
- created_at (timestamp)

Table: interactions
- id (serial primary key)
- user_id (foreign key → users)
- platform (string)
- input_type (text/voice/image)
- user_message (text)
- bot_response (text)
- language_detected (string)
- subject_keywords (text array)
- points_earned (integer)
- response_time_ms (integer)
- created_at (timestamp)

Table: conversation_memory
- id (serial primary key)
- user_id (foreign key → users)
- role (user/assistant)
- content (text)
- created_at (timestamp)
